# Privacy And Sanitization

Event records should preserve evidence without leaking private context.

Before adding material:

- remove secrets, tokens, credentials, and keys;
- remove customer, client, or personal data;
- replace local machine paths with generic paths;
- replace private repository names with generic labels;
- avoid screenshots unless they are fully sanitized;
- avoid unreleased plans unless the repository is private.

When in doubt, summarize the evidence and point to a private source instead of copying
the source itself.
