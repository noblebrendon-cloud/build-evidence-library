# Contributing

Contributions should preserve the core boundary: capture verified evidence first,
create public material only through deliberate later work.

## Guidelines

- Keep examples fictional or fully sanitized.
- Avoid adding dependencies unless the value is clear and durable.
- Include tests for tool behavior.
- Keep templates readable as plain Markdown.
- Do not add external service calls to the scaffold.

## Local Checks

```bash
python -m pytest
```
