# Event Schema

Event IDs use this shape:

```text
EVT-YYYY-MM-DD-short-kebab-name
```

Each event folder contains:

| File | Purpose |
| --- | --- |
| `00_EVENT.md` | Summary, boundaries, status, and source references. |
| `01_EVIDENCE.md` | Verified facts and test evidence. |
| `02_TEACHING_ATOMS.md` | Reusable lessons extracted from the event. |
| `03_DERIVATIVE_BACKLOG.md` | Possible future outputs, not drafts. |
| `04_PUBLICATION_LEDGER.md` | Public outputs that actually exist. |
